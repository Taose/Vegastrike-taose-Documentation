366  // compiler. Specifically, the problem is that the compiler botches code
367  // generation if an unseen method (which is later declared inline) is
368  // called from within another inline method. For instance, if the
369  // constructors were listed at the top of the file, rather than here, the
370  // compiler would see calls to Empty() and StartBoundingBox() before seeing
